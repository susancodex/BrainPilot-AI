# BrainPilot AI 🧠

An AI-powered study companion SaaS that helps students plan, revise, quiz themselves, and track academic progress — built with Django 6 + React 19 and powered by Google Gemini with automatic multi-provider failover.

## Features

| Feature | Description |
|---|---|
| **AI Study Planner** | Generate daily/weekly/emergency study plans from subjects, dates, and syllabus (PDF upload or paste) |
| **Smart Notes** | Rich-text notes with AI summarization and one-click flashcard generation |
| **AI Quizzes** | MCQ, True/False, and short-answer quizzes generated by Gemini; personalised coaching feedback on submission |
| **Spaced Repetition** | Flashcard review system with difficulty tracking and due-date scheduling |
| **Revision Tracker** | Log and track revision topics; surfaces weak areas automatically |
| **Goals** | Set learning goals with progress tracking |
| **AI Chatbot** | Real-time study assistant with SSE streaming word-by-word |
| **Analytics** | Study trends, subject breakdown, performance reports |
| **Pomodoro & Focus Logs** | Built-in Pomodoro timer with streak tracking |
| **PDF Chat** | Upload any PDF and chat with it using AI |
| **Multi-Provider AI Gateway** | Gemini → Groq → OpenRouter automatic failover with health monitoring and prompt security |
| **Notifications** | In-app alert system |

## Getting Started

Both services start automatically when you press **Run** in Replit.

| Service | Port | Workflow |
|---|---|---|
| Frontend (React/Vite) | 5000 | `BrainPilot Frontend` |
| Backend (Django API) | 8000 | `BrainPilot Backend` |

The frontend proxies all `/api/*` and `/media/*` requests to `localhost:8000`.

### Manual backend commands (from `backend/`):

```bash
make run              # start dev server
make migrate          # apply migrations
make makemigrations   # generate new migrations
make shell            # Django shell
make help             # all available commands
```

## AI Gateway

BrainPilot uses a **three-provider failover gateway** so the app never goes down due to a single AI provider outage or quota limit.

```
Request → Gemini (primary)
             ↓ (rate-limit / error)
          Groq (fallback 1)
             ↓ (rate-limit / error)
          OpenRouter (fallback 2, 5-model rotation)
             ↓ (all fail)
          AllProvidersUnavailableError → user-friendly error
```

**Key behaviours:**
- After 3 consecutive failures, a provider enters a 5-minute cooldown
- Prompt injection attempts are blocked before any API call
- `/api/v1/ai/health/` returns live status, success rate, and latency for all providers
- All existing services (`chatbot`, `notes`, `quizzes`, `planner`) route through the gateway transparently via the `GeminiAdapter` shim — no service changes needed

## Architecture Notes

- **Thin views** — all business logic lives in `services.py` per app; all AI logic in `services/ai_engine/`
- **SSE streaming** — `chatbot/send/stream/` uses `StreamingHttpResponse` + `generate_content_stream`. Events: `chunk`, `done`, `error`
- **Celery** — task `.delay()` calls are wrapped in try/except; server degrades gracefully without a Redis broker (expected in dev)
- **PYTHONPATH** — `backend/` is on `PYTHONPATH` so Django imports resolve from the `backend/` root
- **Analytics** — has no models; queries cross-app tables at runtime
- **Notes** — `content` field allows blank strings so new notes can be created empty and filled in later
- **Syllabus-aware planning** — the `extract-syllabus` endpoint extracts PDF text server-side; the planner prompt injects it so AI structures sessions by actual course content

## Development Tips

- Django takes ~5–10s on first boot (app registry + migration check)
- Quiz generation requires `question_count >= 3` (enforced by backend serializer)
- Groq free tier: 12,000 tokens/min — rate limits are transient, not permanent failures
- OpenRouter free models are heavily shared; the gateway rotates through 5 models automatically
- Celery broker warnings in dev are expected and non-fatal
- `DJANGO_SETTINGS_MODULE=config.settings.development` must be set for dev (configured in `.replit`)
