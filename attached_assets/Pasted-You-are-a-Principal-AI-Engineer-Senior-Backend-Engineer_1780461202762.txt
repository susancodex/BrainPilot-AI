You are a Principal AI Engineer, Senior Backend Engineer, Security Engineer, and Production SaaS Architect.

Implement a Production-Grade Multi-Provider AI Gateway for BrainPilot AI.

======================================================================
OBJECTIVE
=========

BrainPilot AI must never depend on a single AI provider.

Implement an intelligent AI routing system that supports:

1. Gemini (Primary)
2. Groq (Secondary Fallback)
3. OpenRouter (Final Fallback)

If one provider becomes unavailable, reaches quota limits, rate limits, or returns errors, the system must automatically switch to the next provider without affecting the user experience.

Users should never know which provider is being used.

======================================================================
SUPPORTED PROVIDERS
===================

PRIMARY

* Google Gemini

FALLBACK 1

* Groq

FALLBACK 2

* OpenRouter

======================================================================
ENVIRONMENT VARIABLES
=====================

Store all secrets securely in environment variables.

GEMINI_API_KEY=

GROQ_API_KEY=

OPENROUTER_API_KEY=

Never hardcode API keys.

Never expose API keys to the frontend.

Never commit API keys to Git.

======================================================================
ARCHITECTURE REQUIREMENTS
=========================

Create a dedicated AI architecture.

backend/
├── ai/
│
├── providers/
│   ├── gemini_provider.py
│   ├── groq_provider.py
│   └── openrouter_provider.py
│
├── gateway.py
├── factory.py
├── services.py
├── exceptions.py
├── interfaces.py
└── types.py

Each provider must be isolated.

Provider logic must not be mixed with business logic.

======================================================================
AI GATEWAY SYSTEM
=================

Create a centralized AI Gateway.

Frontend should never directly call Gemini, Groq, or OpenRouter.

Frontend must call:

/api/ai/chat

The AI Gateway decides which provider to use.

Example Flow:

Attempt Gemini

If Gemini succeeds:
return response

If Gemini fails:
automatically try Groq

If Groq fails:
automatically try OpenRouter

If OpenRouter fails:
return graceful system error

======================================================================
FAILOVER CONDITIONS
===================

Automatically switch providers when:

* Quota exhausted
* Rate limited
* API unavailable
* Timeout
* Network failure
* Service outage
* Invalid response
* Authentication error

The system must never crash.

The user must never see raw provider errors.

======================================================================
PROVIDER HEALTH MONITORING
==========================

Track:

* Success rate
* Failure rate
* Average latency
* Availability

Temporarily avoid providers with repeated failures.

Automatically restore them when healthy.

======================================================================
PROMPT SECURITY
===============

Validate prompts before sending.

Protect against:

* Prompt injection
* Malicious inputs
* Excessively large prompts
* Empty prompts

Sanitize user input where appropriate.

======================================================================
ERROR HANDLING
==============

Create standardized errors.

Do not expose:

* Stack traces
* Internal exceptions
* API responses
* Secrets
* API keys

Return user-friendly errors only.

======================================================================
LOGGING

* Failover events

Never log:

* Sensitive information

PERFORMANCE
===========

Implement:

* Request timeout protection
* Retry logic
* Exponential backoff
* Connection reuse
* Response caching where useful

Avoid unnecessary AI calls.
